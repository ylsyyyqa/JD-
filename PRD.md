---

## 一、产品概述

### 1.1 产品定位

**"用AI把岗位JD翻译成你的成长路线图"**

JD智析是一个面向实习生/应届生的AI求职决策工具。不只是一个"JD vs 简历"对比工具，而是一个从迷茫到行动的决策引擎——告诉你差在哪，更告诉你缺的东西怎么从零补上。

### 1.2 目标用户

| 优先级 | 用户画像 | 核心痛点 |
|--------|---------|---------|
| P0 | 大三/研二找实习的学生 | 看不懂JD真实要求，简历空空不知从何开始 |
| P0 | 应届生秋招/春招 | 海投无回音，想知道自己到底差在哪 |
| P1 | 转行新人（0-2年经验） | 不知现有经验哪些可迁移，哪些要补 |
| P2 | 大一大二提前规划的学生 | 看到JD要求一脸懵，想提前准备但没方向 |

### 1.3 产品边界（MVP明确不做的事）

- **不做**：社招跳槽（3年+经验人群）
- **不做**：行业未来趋势分析 & 入局发展推演（Roadmap P2）
- **不做**：实时爬取招聘平台数据
- **不做**：在线简历编辑/排版（复用 resume-builder skill 的能力）

## 二、核心功能

### 模块 A：JD-简历缺口分析引擎

#### A1. 输入

| 输入项 | 支持格式 | 处理方式 |
|--------|---------|---------|
| 岗位JD | 文本粘贴 / PDF / 截图(OCR) | AI结构化提取 |
| 个人简历 | 文本粘贴 / PDF / Word / 截图(OCR) | AI结构化提取 |
| Demo体验 | 预设3组示例JD+简历 | 一键填充，零门槛体验 |

#### A2. AI处理流程

```
┌──────────┐    ┌──────────┐
│  JD文本   │    │  简历文本  │
└────┬─────┘    └─────┬────┘
     ↓                ↓
  结构化提取        结构化提取
  · 硬技能要求      · 硬技能清单
  · 软技能要求      · 软技能证据
  · 经验要求        · 经历列表
  · 学历要求        · 学历信息
  · 加分项          · 项目/作品
     │                │
     └───────┬────────┘
             ↓
       逐维度对比打分
    （完全匹配 / 部分匹配 / 缺失）
             ↓
       缺口严重度分级
    🔴 致命缺口 — JD硬门槛不满足
    🟠 重要缺口 — 核心技能缺失
    🟡 加分缺口 — 加分项缺失
             ↓
       加权匹配度计算
    硬技能50% + 软技能20%
    + 经历20% + 学历10%
             ↓
       生成缺口报告
```

#### A3. 统一能力标签体系（全岗位通用抽象）

不区分"技术岗"和"非技术岗"，所有岗位统一抽象为四维能力模型：

```
能力标签体系（面向实习生/应届生）

A. 硬技能（可验证的工具/方法）
   技术类：编程语言、框架、数据库、工具（Git/Docker/Figma...）
   业务类：数据分析、用户调研、竞品分析、SQL、需求文档...
   设计类：Figma、PS、用户旅程图、线框图、设计系统...

B. 软技能（通过经历体现）
   沟通表达、团队协作、项目管理、逻辑思维、主动性、学习能力...

C. 经历证据（你做过的事）
   实习经历、校园项目、竞赛获奖、社团活动、个人作品、课程设计...

D. 知识领域（你懂什么）
   电商、SaaS、内容社区、B端、金融科技、教育、游戏、AI...
```

#### A4. 岗位类型 × 技能维度矩阵（复用 resume-builder 分类体系）

### 模块 B：从0开始的项目思路生成

#### B1. 技术岗 — 三级递进代码项目

| 级别 | 周期 | 目标 | 产出物 |
|------|------|------|--------|
| L1 入门 | 1-3天 | 掌握核心API/基础用法 | 可运行的Demo |
| L2 综合 | 1-2周 | 串联2-3个技能点 | 完整小项目 |
| L3 深度 | 1个月 | 可写入简历的作品 | 完整项目+文档 |

每个项目附带：技术栈清单 / 核心功能列表 / 学习资源关键词 / 简历话术建议

#### B2. 非技术岗 — 三级递进作品集项目

| 岗位类型 | L1（1-2天） | L2（1周） | L3（2-4周） |
|---------|-----------|---------|-----------|
| 产品经理 | 产品拆解报告 | PRD实战文档 | 完整竞品分析+低保真原型 |
| 运营 | 爆款内容拆解 | 运营方案策划 | 真实账号运营实验+复盘 |
| 市场营销 | 营销案例分析 | 营销活动方案 | 真实推广实操+数据复盘 |
| UI/设计 | 设计规范复刻 | App页面改版方案 | 从0设计完整虚拟产品 |
| 数据分析 | 公开数据集分析 | 完整分析报告 | 数据看板/预测模型 |

## 三、信息架构 & 页面结构

```
首页 (/)
├── Hero区：一句话价值主张 + CTA按钮
├── 3步流程说明（上传→分析→行动）
├── Demo体验入口（一键加载示例数据）
└── 页脚

上传页 (/upload)
├── 步骤1：上传/粘贴JD
│   └── 支持文本/PDF/截图
├── 步骤2：上传/粘贴简历
│   └── 支持文本/PDF/Word/截图
├── 步骤3：选择岗位类型（可选，AI自动识别）
└── [开始分析] 按钮

缺口报告页 (/report/:id)
├── 顶部：匹配度大数字 + 进度环
├── 中部：五维雷达图
├── 下部：三级缺口卡片列表
│   ├── 🔴 致命缺口
│   ├── 🟠 重要缺口
│   └── 🟢 已有优势
├── 底部：一句话总结
└── 行动按钮：[生成项目思路] [导出报告]

├── 按缺口分组的项目卡片
├── 三级难度筛选（L1/L2/L3）
├── 每个卡片可展开查看详情
└── [导出全部项目思路]
```

## 四、非功能需求

### 4.1 性能

- 首屏加载 < 3s
- 分析结果生成 < 15s（含AI调用）
- 雷达图渲染 < 1s

### 4.2 兼容性

- 支持 Chrome / Edge / Safari 最新两个版本
- 移动端响应式（报告页可读，编辑操作建议桌面端）

### 4.3 安全

- 用户上传的JD和简历不持久化存储（分析后自动删除）
- 传输层HTTPS加密
- 不要求注册即可使用核心功能

### 4.4 数据

- Demo模式使用预设脱敏数据
- AI分析结果不用于模型训练
- 用户主动导出的报告由用户自行保管
